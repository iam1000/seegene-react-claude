# 6.3변경된 기능

1. **Input 계열(Grid celledit 포함) 컴포넌트의 입력폼에서 Ctrl 키를 누른 상태에서 왼쪽/오른쪽 방향키를 누르면, 커서가 단어 단위로 이동하도록 변경했습니다. (RP 102560, RP 102509)**
 

| 설명 |
| --- | --- |
| NRE 와 WRE의 포커스(Focus) 동작이 상이했던 내용을 NRE에서 WRE의 동작과 동일하도록 수정한 작업입니다. | |
| 변경 전 | Ctrl 키를 누르고 방향키를 누르면, 커서가 한 글자씩 이동 |
| 변경 후 | Ctrl 키를 누른 상태에서 왼쪽/오른쪽 방향키 누르면, 커서가 단어 단위로 이동 |

2. **Radio 컴포넌트의 Key Action에 대해 포커스 이동과 아이템 결정을 변경했습니다. (RP 103178, RP 104107)**
 

| 설명 |
| --- | --- |
| 변경 전 | Radio 컴포넌트에 포커스가 있을 때 Space 키를 입력해도 아이템이 선택되지 않음 |
| 변경 후 | Radio 컴포넌트의 index 속성값이 -1 인 경우에는 이동한 위치의 아이템을 Space 키 입력으로 선택 가능 |
| 자세한 내용은 "Nexacro N Reference Guide"를 확인하시기 바랍니다.Components > Component > Radio > Basic Key ActionComponents > Component > Radio > Accessibility Key Action | |

3. **컴포넌트 삭제 후 Undo(Ctrl+Z) 키 입력으로 컴포넌트 복원 시 컴포넌트가 선택 상태가 되도록 변경했습니다. (RP 104032)**
 

| 설명 |
| --- | --- |
| 사용자가 Delete 키 또는 우클릭 삭제 메뉴를 통한 컴포넌트의 '삭제' 후 Undo 동작에서만 적용되는 제한적 기능입니다. | |
| 변경 전 | Undo(Ctrl+Z) 동작으로 복원된 컴포넌트가 선택 상태가 아님 |
| 변경 후 | Undo(Ctrl+Z) 동작으로 컴포넌트가 선택된 상태로 복원됨 |

4. **nodatatext 속성값을 설정하고 Grid 컴포넌트에 표시할 데이터가 없을 때, Grid Body Band의 accessibilityenable 속성값에 따라 nodatatext 속성값의 처리 방식이 달라지도록 변경했습니다. (RP 103384)**
 

| 설명 |
| --- | --- |
| 변경 전 | 방향키로 그리드 처음 접근 시 nodatatext 속성값을 읽음 |
| 변경 후 | accessibilityenable 속성값에 따라 nodatatext 속성값의 처리 방식이 다름 |
| 자세한 내용은 "Nexacro N Reference Guide"를 확인하시기 바랍니다.Components > Component > Grid > Property > nodatatext > 접근성 설정 시 동작 | |

5. **안드로이드 정책 변경으로 64비트 기기 대상 SO 파일 빌드 시 16KB 페이지 크기를 지원하도록 변경했습니다. (RP 103012)**
 

| 설명 |
| --- | --- |
| 변경 전 | Android에서 4KB 메모리 페이지 크기만 지원 |
| 변경 후 | Android 64비트 대상 SO 파일 빌드 시 16KB 페이지 크기 옵션 지원 |
| 자세한 내용은 다음의 링크를 확인하시기 바랍니다.https://developer.android.com/guide/practices/page-sizes?hl=ko | |

6. **Windows 11 환경에서 32비트 모듈 (SetupRuntime 포함)을 다운로드할 수 있도록 변경했습니다. (RP 102850, RP 102879)**
 

| 설명 |
| --- | --- |
| 변경 전 | "Windows 11" 운영체제 설정이 64비트 모듈 (SetupRuntime 포함)만 설정 가능 |
| 변경 후 | Windows x86 / Windows x64 / Module 파일 32bit/64bit 선택 상태에 따라 산출물 변경 |

7. **BIGDECIMAL 자료형에 대한 최소값~최대값 표현 범위를 변경했습니다. (RP 101005)**
 

| 설명 |
| --- | --- |
| 변경 전 | 정수부+소수부 전체 합쳐서 24자리수로 표현 (소수점 최대 15자리수) |
| 변경 후 | 정수부+소수부 전체 합쳐서 31자리수로 표현 (정수부 최대 24자리, 소수부 최대 15자리수) |
| 자세한 내용은 다음을 확인하시기 바랍니다.Nexacro N Reference GuideData Objects > ColumnInfo > Property > type > Setting Syntax > BIGDECIMALScript > Nexacro Objects > DecimalNexacro N 온라인 응용 개발 가이드Dataset XMLDataset SSVDataset JSON | |

8. **accessibilitylabel에 설정값이 없는 경우 default 값이 없도록 변경했습니다. (RP 99890)**
 

| 설명 |
| --- | --- |
| 변경 전 | accessibilitylabel에 설정값이 없는 경우, default로 Component의 text 또는 value를 설정 |
| 변경 후 | accessibilitylabel에 설정값이 없는 경우, default 값 없음 |
| 다음의 내용이 "Nexacro N Reference Guide" 에서 삭제 되었습니다.컴포넌트 > accessibilitylabel Remark 설명 중 아래 문구"accessibilitylabel 속성값을 설정하지 않더라도 컴포넌트의 name, value, text 속성값을 설정한 경우 해당 속성값이 스크린 리더의 음성 출력 대상으로 포함할 label 정보로 적용됩니다." | |

9. **open 메서드 실행으로 열린 팝업창에서 F5키 입력 시 onkeydown, onkeyup 이벤트가 발생하도록 변경했습니다. (RP 102119)**
 

| 설명 |
| --- | --- |
| 변경 전 | Modeless 팝업창에서 F5키를 입력 시 onkeydown, onkeyup 이벤트가 발생하지 않고 팝업창을 닫았습니다. |
| 변경 후 | Modeless 팝업창에서 F5키를 입력 시 팝업창을 유지하며 onkeydown, onkeyup 이벤트가 발생합니다. |
| 자세한 내용은 다음의 링크를 확인하시기 바랍니다.https://docs.tobesoft.com/product_information_nexacro_n_ko/product_restrictions#64a13ae09933f694 | |
